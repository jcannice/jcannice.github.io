NYC Taxi Ride Duration Predictor

How do we beat rush hour?

Overview

In my Techniques of Data Science class, I got to query a SQLite database of NYC Taxi data to build a regression model that predicts ride length using pick-up time, pick-up and drop-off locations, and distance.


Extracting & Cleaning the Data

To limit the data to trips of ordinary length (1 minute - 1 hour), speed, and positive passenger count, I queried the database using SQL statements. Since most trips took place in Manhattan, I filtered the data based on long/lat coordinates that were contained within the polygon as shown here.
Of the original 100000 taxi trips, 3555 anomolous trips 0.03555 were removed through data cleaning. The data set was further reduced to 82800 for analysis on taxi trips taken within Manahattan.


To weed out any potential anomalies in the data, I created a date feature from the pickup date time and plotted the passenger count in January 2016. The low passenger count days during the 2016 winter blizzard were removed to avoid anomalies in the training data.


EDA & Feature Engineering

I first compared the distributions of taxi ride durations for each day and found that trips tend to be longer during weekdays as opposed to weekends. Thus, I added additional columns for day, weekend, pickup hour, period, and speed.

I then created an overlaid histrogram to compare the distribution of average speeds for taxi rides that start in morning, day, and night. I discovered that speed is generally associated with time of day.

Knowing that Mahattan can be split into three general regions, I employed PCA to divide the trips into three groups: Lower Manhattan, Midtown, and Upper Manhattan.


Model Selection & Performance

I fit the training data to several linear regression models and compared the Root Mean Squared Error between them.

Model using only distance of ride: ~276.78
Model using all features, no regularization: ~255.19
Model that selects paramters based on period: ~246.63
Model that first predicts speed then duration: ~243.02
Model using tree regression errors over combination of period, region, and weekend: ~233.63

It appears tree regression led to a better use of categorical varaibles than linear regression. Domain knowledge of duration as just a function of distance and speed allowed for better prediction when first predicting speed.
